VT 技术学习 (一) 检测支持与VMXON

发表于 2021-12-09 更新于 2021-12-13 分类于 vmx

前言

对于部分反作弊对R3注入的强检测,R0层的技术已经无力应对了,必须得学会hypervisor了

本次学习主要参考bilibili周壑老师的教程,hyperplatform代码,书籍《处理器虚拟化技术》

VT框架流程

  • 本人の粗略理解: 相比于直接用cpu模拟出的VMM,VT技术能使得Guest原生运行大部分指令,让虚拟机的性能得到巨大提升,并且能让虚拟机支持一些CPU的硬件特性

检测CPU是否能使用VT指令集

在此之前先需要了解一下SMX是个什么东西

SMX

  • SMX是面向 intel TXT(可信执行技术) 平台的编程接口,主要指令是GETSEC
  • SMX指令集由 IA32_FEATURE_CONTROL MSR 决定是否开启,该MSR是不可写的

相关资料:

  1. intel手册 volume2 CHAPTER 6 SAFER MODE EXTENSIONS REFERENCE

  2. Intel® Trusted Execution Technology Hardware-based Technology for Enhancing Server Platform Security

IA32_FEATURE_CONTROL 结构

检测支持

  1. 执行cpuid(eax=1)指令,返回的ecx第5位为1说明cpu支持vt指令集
  2. 检测IA32_FEATURE_CONTROL MSR的第0位(lock bit),如果该位是1说明主板bios中开启了vt功能
  3. 检测IA32_FEATURE_CONTROL MSR的第1位,如果为1,则允许在SMX指令GETSEC[SENTER]和GETSEC[SEXIT]之间使用VMX相关指令,执行VMX指令时会受到TXT技术的保护
  4. 检测IA32_FEATURE_CONTROL MSR的第2位,如果为1,则允许在SMX指令GETSEC[SENTER]和GETSEC[SEXIT]之外使用VMX相关指令

VMXON

  • vmxon需要一块4kb内存来记录维护虚拟机状态,称为VMXON region

VMXON初始化过程

  1. 为VMXON region申请一块4KB的内存,首4字节的[30:0]写入VMCS revision identifier,[31]=0
  2. 使用IA32_VMX_CR0_FIXED0 (index 486H) 和 IA32_VMX_CR0_FIXED1 (index 487H) 调整CR0
  1. 使用IA32_VMX_CR4_FIXED0 (index 488H) 和 IA32_VMX_CR4_FIXED1 (index 489H) 调整CR4

  1. 执行vmxon指令,参数传入VMXON region的物理地址,如果指令执行之后RFLAGS.CF = 0,则VMXON初始化成功

  2. vmxon执行之后cpu进入vmx操作模式,期间CR4.VMXE不可设为0,否则会触发#GP,可以用vmxoff指令退出vmx操作模式,如果没有执行到vmlanuch,不会对系统正常运行造成影响

相关资料: intel手册 volume3 31.5 VMM SETUP & TEAR DOWN
